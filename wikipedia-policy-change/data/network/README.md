# Policy core network — 6 wikis (2026)

The current (2026) policy/guideline **core** of six Wikipedia editions, as a single graph.
Generated by [`../../net/build_network.py`](../../net/build_network.py) (repeatable).

- **Nodes** = core pages across en/de/nl/fr/es/ja. Node id = `<wiki>:<page_id>`.
- **Edges** =
  - `wikilink` — intra-wiki in-body links, **core→core** (directed), parsed from wikitext.
  - `interwiki` — cross-language links taken **directly from the replica `langlinks` table**
    (no Wikidata), kept only when the target is itself a core node on the target wiki.

Build (2026): **1,143 nodes**, **14,106 edges** (12,617 wikilink · 1,489 interwiki).
Per-wiki core: en 347, ja 193, fr 177, de 149, es 148, nl 129.

## Files

`nodes.csv`
| column | meaning |
|---|---|
| node_id | `<wiki>:<page_id>` |
| wiki, page_id, title, namespace | identity |
| qid | Wikidata item (informational; **not** used for the interwiki edges) |
| confidence | `core` |
| admitted_via | status_template / overview / iw_qid / scored_* |
| native | 1 = surfaced by this wiki's own signal (banner/overview); 0 = via cross-wiki propagation |
| status_tier | policy/guideline/essay/… where known (en only so far) |

`edges.csv` — `source, target, type (wikilink|interwiki), wiki` (wiki blank for interwiki).

`policy_network.graphml` — same graph for Gephi/Cytoscape/networkx; **not committed**
(regenerable; ~2.6 MB). Recreate with `build_network.py`.

## Regenerate (on Toolforge)

```
toolforge jobs run net-graph --image python3.13 --mem 2Gi --wait \
  --command "~/venv/bin/python -u ~/wikimedia-analysis/wikipedia-policy-change/net/build_network.py --year 2026"
```

## Caveats

- de/nl/fr/es/ja core has no per-page essay demotion yet (en-only banner list), so `status_tier`
  is mostly null and a few non-policy pages may persist in those cores.
- Interwiki edges are **direct langlinks**, so a core page lacking a langlink to another core
  page contributes no cross-wiki edge even if a conceptual equivalent exists — that gap is the
  signal the content-alignment step (M9) investigates, not a defect here.
